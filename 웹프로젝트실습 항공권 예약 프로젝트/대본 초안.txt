========================== 기획의도 ==========================

여행에 관심이 많은 사람끼리 여행에 대한 정보를 공유하고 소통하며 보다 좋은 여행 계획을 세울 수 있고, 항공권 예약과 관광지 정보를 동시에 제공하여 더욱더 쉽고 편리하게 여행을 즐길 수 있다. 서로의 여행을 공유하며 여행에 대한 좋은 추억을 쌓는 것을 돕기 위해 이 사이트를 기획하게 되었다.
벤치마킹: 아시아나 항공, 대한항공 페이지 벤치마킹 하였다.
주요 고객: 여행을 즐기고 여행에 대한 정보를 필요로 하는 20~40대 학생이나 직장인


========================== 주요 기능 ==========================

항공권 검색/예약/가격비교 기능
관광명소/맛집 추천 및 리뷰 기능
관광명소/맛집 저장 기능
나의 리뷰 수정 및 관리 기능
나의 예약 조회 및 관리 기능
관리자 - 회원/예약/리뷰/저장 관리 기능
>> 이 중 항공권 조회/예약, 관광명소/맛집 저장, 리뷰/예약 관리, 관리자 메뉴 구현


========================== DB 테이블 소개 ==========================

회원, 공항, 비행기, 좌석, 예약좌석, 일정, 예약, 상품, 장바구니 테이블 존재
1. 회원: 말 그대로 회원 정보 저장, 로그인, 회원가입, 회원정보관리에서 접근
2. 공항: 공항 번호, 공항 이름, 국가 저장
3. 비행기: 비행기 번호, 비행기 이름, 비행기 총 좌석 수 저장
4. 좌석: 좌석번호, 비행기 번호, 등급, 좌석 수 저장,
	아시아나 항공사의 AA번 비행기를 예로 들면,
	이코노미는 450개, 비즈니스는 75개, 퍼스트는 25개의 좌석이 있음을 저장
5. 예약좌석: 예약좌석번호, 좌석번호, 비행기 번호 저장
	마찬가지로 아시아나 항공사의 AA번 비행기를 예로 들면,
	A열의 1번 좌석번호는 AA01, 2번 좌석번호 AA02 등의 정보 저장되어있음
6. 일정: 일정번호, 출발시간, 도착시간, 출발지, 도착지, 비행기번호, 가격 저장
7. 예약: 회원이 예약한 스케줄, 좌석번호 등의 정보 저장
	예약번호, 회원아이디, 탑승일, 일정번호, 좌석번호, 운임료 등의 필드가 있음
8. 상품: 관광명소와 맛집의 정보가 저장되어있는 테이블,
	여행지추천, 관광명소/맛집 저장 기능 등에서 접근
	상품번호, 상품이름, 위치, 상품상세, 평점, 리뷰수, 
	상품구분, 이용시간, 전화번호, 이미지경로 등의 필드 존재
9. 장바구니: 내가 저장한 관광명소/맛집의 정보 저장
	장바구니번호, 상품번호, 회원아이디 필드 존재


========================== 페이지 소개 ==========================

1. 메인 > 대한항공 페이지 디자인 많이 참조 (캐러셀 보여주기 - 점부터 누르기)
top.jsp, content.jsp, bottom.jsp로 모듈화
여행지 추천이 메인에 뜸 > (여행지 추천 or 항공권 예약) 클릭 > 로그인 이후 접근 가능
항공권 예약, 마이페이지는 로그인 이후 접근 가능하게 해놓음, 로그인 페이지로 자동 이동

2. 로그인 페이지
DB와 연결하여 비밀번호가 틀리면 비밀번호가 틀렸다는 얼럿창 출력
아이디와 비밀번호 저장 가능
로그인 완료하면 메인 페이지로 이동하고 ㅇㅇ님, 로그아웃으로 top.jsp의 내용이 바뀜
로그아웃하면 다시 로그인으로 top.jsp의 내용이 바뀜

3. 회원가입 페이지
DB와 연결하여 중복된 아이디로 회원가입하는 경우 이미 존재하는 아이디라는 얼럿창 출력
회원가입 중 어느 한 필드의 정보 입력이 누락되었을 경우
해당 값을 입력해달라는 얼럿창 출력, 커서가 해당 필드로 이동
회원가입 완료 시 회원가입이 완료되었다는 얼럿창 출력, 로그인 페이지로 이동

4. 항공권 예약 페이지
1) reservationForm.jsp: 출발지와 도착지 등의 정보를 입력하고 확인
2) reservationProcess1.jsp: 
- 가격순, 출발시간 순 등으로 일정 정보 정렬 가능
- 일정 번호를 클릭하면 해당 일정으로 예약 가능
3) reservationProcess2.jsp: 
- 좌석을 클릭하면 좌석 선택 페이지로 이동
- 이미 선택된 좌석일 경우 선택 불가
- 일정, 그리고 등급 별로 가격 책정되어 ticket_price에 저장됨
- 아까 선택했던 날짜, 출발지, 도착지, 항공사 이름 등의 예약 정보 확인하고 예약

7. 나의 예약 페이지(myReservation.jsp)
DB와 연결하여 예약, 일정, 예약좌석, 좌석 총 4개의 테이블을 조인하여 예약 정보 확인 가능
예약번호를 누르면 예약 변경과 예약 취소 가능
예약 취소를 누르면 delete 실행, 다시 나의 예약 페이지로 이동

8. 마이페이지
1) myPage.jsp: 내가 예약한 숙박 정보 확인 가능
2) myReservation.jsp: 아까 확인했듯이 나의 항공 예약 정보 확인 가능
3) myInformation.jsp: DB와 연결되어 나의 회원 정보 확인 및 수정 가능
4) myReview.jsp: 내가 작성한 리뷰 목록 확인 가능
5) myList.jsp: 내가 저장한 관광명소/맛집 목록 확인 및 수정 가능
- DB와 연결하여 상품, 장바구니 총 2개의 테이블을 조인하여 찜 정보 확인 가능

9. 여행지 정보 페이지
관광명소/맛집별로 하트를 눌러 찜 가능
찜한 관광명소/맛집은 나의 찜목록 페이지에서 확인 가능

10. 관리자 페이지: 회원정보, 예약, 찜 관리 등 페이지 관리 가능
- 아이디, 예약번호, 찜목록번호를 누르면 수정/삭제 가능
- 수정/삭제하기 위해서 관리자 비밀번호 필요
- 저장/취소/삭제 중 취소 누르면 수정했던 내용 원래대로 돌아감
- 저장/취소/삭제 중 삭제 누르면 정말로 삭제하시겠습니까? 얼럿창 뜨고 확인 누르면 delete 수행
1) 회원정보관리: 아이디를 누르면 해당 아이디의 회원 정보 수정
2) 예약관리: 모든 회원들의 예약 정보 관리
3) 찜목록관리: 모든 회원들의 관광명소/맛집 저장 목록 관리
